# 示例 2（SaaS/B2B）：租户权限管理

## 原始需求

> 平台要加权限管理，不同人有不同权限。

---

## Step 0 诊断

评分：
- G=1
- S=0
- C=0
- R=1
- E=1（已有客户投诉）
- T=3

判定：
1. 领域：saas-b2b
2. 路由：Discovery -> PRD 博弈
3. 不满足高分直出条件

---

## Step 2 追问

### 第 1 轮（3 个 P0）

1. 痛点是"看到了不该看的数据"还是"做了不该做的操作"？
2. 管理员是平台管理员还是租户管理员？
3. 最近一次权限事故是什么？

回答摘要：
- 普通员工看到了薪资数据
- 每个租户自己的管理员
- 出现过敏感数据导出事件

### 第 2 轮（3 个 P0）

1. 权限粒度到哪层（页面/操作/数据范围）？
2. 角色定义方式（平台预设/租户自定义/混合）？
3. 离职后权限如何回收？

回答摘要：
- 三层都要
- 平台预设 + 租户微调
- 与企业微信状态同步自动关权

### 第 3 轮（2 个 P0）

1. 临时授权是否带有效期并自动回收？
2. 是否需要完整审计日志？

回答摘要：
- 需要，按时间自动收回
- 需要

---

## Step 3 收敛确认

关键结论：
1. 租户级权限模型（非平台级）
2. 三层粒度权限
3. 临时授权 + 到期回收
4. 审计日志刚需

待确认：
1. 自定义数据范围是"选人"还是"选组织单元"
2. 变更通知是否必须发送给被授权人

---

## 推荐模板组合

核心通用模板 + 扩展包 A（权限）+ 扩展包 B（策略）
