## 1. Wiki 概述

### 1.1 定义

Wiki 是一种基于互联网的**协作式超文本系统**，允许用户通过浏览器直接创建、编辑和修改网页内容，所有页面通常存放在同一个数据库中，支持多人协同编辑和版本控制。

### 1.2 词源与起源

- **名称来源**："Wiki"一词来源于夏威夷语 **"wiki wiki"**，意为"快速"或"快点"
- **创建者**：由 **Ward Cunningham** 于 **1995 年** 创建
- **核心理念**：体现该技术**快速创建网页**的核心理念

### 1.3 核心特点

| 特点 | 说明 |
|------|------|
| **开放编辑** | 基于"善意假设"原则，所有访问者都可参与建设 |
| **版本控制** | 追踪每次修改，支持回滚和历史记录查看 |
| **多人协作** | 支持团队成员同时编辑同一页面 |
| **快速迭代** | 无需掌握 HTML 即可参与内容创建 |

### 1.4 技术架构

```
用户浏览器 ←→ HTML 显示 ←→ Wiki 引擎 ←→ 后端数据库
                              ↑
                    (PHP/Python/Ruby等服务器端脚本)
```

## 2. 核心功能

### 2.1 内容管理

- **版本控制**：自动保存每次修改，支持版本对比和回滚
- **历史记录**：完整记录所有编辑操作，可追溯
- **页面链接与反向链接**：支持双向链接关系

### 2.2 协作功能

- **讨论页面**：每个条目配有专属讨论区供编辑者沟通
- **监视列表**：用户可监视感兴趣的页面，修改时收到通知
- **编辑摘要**：每次保存需填写修改说明

### 2.3 搜索与导航

- **全文搜索**：支持关键词检索
- **目录导航**：页面自动生成目录，快速跳转
- **模糊搜索**：支持自动补全和近似匹配

### 2.4 扩展能力

- **模板系统**：可复用标准格式和内容块
- **插件机制**：支持扩展安装以增强功能
- **多语言支持**：内置国际化功能

## 3. 企业应用场景

### 3.1 典型应用

1. **团队文档协作与版本管理**
2. **项目知识沉淀与共享**
3. **新员工培训资料库建设**
4. **企业流程规范的在线维护**
5. **打破信息孤岛、实现跨部门知识关联**
6. **支持页面间的双向链接、知识图谱构建**

### 3.2 优势与挑战

| 优势 | 挑战 |
|------|------|
| 打破信息孤岛 | 学习曲线较陡 |

| 双向链接、知识图谱 | 需要持续维护 |
| 集中存储、统一管理 | - |

## 4. 主流 Wiki 系统对比

### 4.1 系统概览

| 系统 | 技术栈 | 适用场景 | 特点 |
|------|--------|----------|------|
| **MediaWiki** | PHP + MySQL | 技术文档/大型组织 | 功能强大、可扩展、开源免费 |
| **DokuWiki** | PHP（纯文本） | 中小团队 | 轻量、无需数据库、迁移简单 |
| **Confluence** | Java | 企业协作 | 与 Jira 深度集成、付费企业级 |
| **Notion** | 云端 | 灵活协作 | 界面现代化、免费个人版 |

### 4.2 选型建议

| 需求 | 推荐 |
|------|------|
| **技术文档首选** | MediaWiki / DokuWiki |
| **企业协作首选** | Confluence / Notion |
| **无需数据库** | DokuWiki |
| **深度 Jira 集成** | Confluence |

### 4.3 MediaWiki 特性

MediaWiki 是维基百科所使用的开源 Wiki 引擎：

- ✅ 高度可扩展性和强大的版本控制功能
- ✅ 支持多语言、模板系统和丰富的插件生态
- ✅ 免费开源，可自定义扩展
- ❌ 配置复杂，对新手不友好
- ❌ 默认界面较为简陋
- 📊 被全球数万个网站和数千家企业使用

## 5. 编辑指南

### 5.1 基本操作

#### 浏览与搜索
- 使用搜索框输入关键词查找条目
- 条目页面包含目录导航，可快速跳转各章节
- 左侧边栏提供"阅读"、"编辑"、"查看历史"等功能链接

#### 账号权限

| 角色 | 权限 |
|------|------|
| 匿名用户 | 浏览、简单编辑 |
| 注册用户 | 创建新词条、参与讨论、监视列表、上传文件 |

### 5.2 常用编辑语法

```wiki
<!-- 链接 -->
[[条目名]]

<!-- 标题 -->
== 二级标题 ==
=== 三级标题 ===

<!-- 列表 -->
* 无序列表项
# 有序列表项

<!-- 格式 -->
'''粗体文本'''
''斜体文本''

<!-- 水平线 -->
----
```

### 5.3 编辑流程

1. 点击"编辑"使用可视化编辑器，或"编辑源代码"使用 Wikitext
2. 编写内容
3. 点击"显示预览"检查效果
4. 填写编辑摘要说明修改内容
5. 保存页面

### 5.4 编辑规范（以 Wikipedia 为例）

| 原则 | 说明 |
|------|------|
| **中立观点 (NPOV)** | 内容应保持中立，不偏袒任何观点 |
| **可验证性** | 所有内容需标注参考资料 |
| **非原创研究** | 不添加未经证实的研究内容 |

## 6. 相关项目

### Wikidata 维基数据

- **定义**：免费开放的结构化知识库
- **作用**：作为 Wikimedia 项目的结构化数据中心，为 Wikipedia、Wiktionary、Wikibooks 等提供支持
- **规模**：截至 2025 年，拥有超过 **1.2 亿**个数据实体
- **特点**：数据采用免费许可证，可导出为标准格式，能与其他开放数据集互联
