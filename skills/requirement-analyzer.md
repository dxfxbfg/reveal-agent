# requirement-analyzer Wiki

## 需求分析模式库

### 典型需求类型

**技术分享型**
- 受众：技术人员
- 风格：Technical blueprint / Academic minimalism
- 色调：冷色调（蓝/灰/青），克制
- 情绪：专业、权威

**商业路演型**
- 受众：投资人/管理层
- 风格：Luxury report / Editorial
- 色调：深色系（深蓝/金/白），高端感
- 情绪：自信、有说服力

**教学培训型**
- 受众：学员/内部团队
- 风格：Playful explainer / Swiss modernism
- 色调：温暖（橙/绿），友好
- 情绪：轻松、鼓励

**学术报告型**
- 受众：学术同行/导师
- 风格：Academic minimalism
- 色调：纯色/单色，严谨
- 情绪：客观、精确

### 信息收集清单模板

1. 主题核心内容（是什么？）
2. 目标受众背景（给谁看？）
3. 关键数据/案例（有哪些证据？）
4. 参考资料/链接（有哪些现成素材？）
5. 演示时长要求（多久？）
6. 特殊要求（数学公式？代码？图表？）
