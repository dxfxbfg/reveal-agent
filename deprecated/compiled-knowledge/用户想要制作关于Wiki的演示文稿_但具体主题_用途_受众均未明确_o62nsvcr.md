## 1. 定义与起源

### 1.1 Wiki 的定义

Wiki 是一种基于互联网的**协作式超文本系统**，允许用户通过浏览器直接创建、编辑和修改网页内容。所有页面通常存放在同一个数据库中，支持多人协同编辑和版本控制。

### 1.2 名称来源

"Wiki"一词来源于夏威夷语 **"wiki wiki"**，意为**"快速"**或**"快点快点"**。1995年 Ward Cunningham 选择这个名字，体现了该技术快速创建网页的核心理念。

### 1.3 创立背景

| 项目 | 详情 |
|------|------|
| 发明者 | Ward Cunningham（沃德·坎宁安） |
| 创建时间 | 1995 年 |
| 首个应用 | 波特兰模式知识库（Portland Pattern Repository） |
| 技术理念 | 基于"善意假设"原则的开放协作 |

## 2. 核心概念与特点

### 2.1 技术架构

Wiki 系统基于**"编辑-保存"模式**，用户无需掌握 HTML 即可参与内容创建。

### 2.2 核心特点

| 特点 | 说明 |
|------|------|
| **开放编辑** | 基于"善意假设"原则，所有访问者都可参与建设 |
| **版本控制** | 完整的修订历史记录，支持回溯和比较 |
| **多人协作** | 允许多人同时编辑同一页面 |
| **快速迭代** | 内容更新即时生效，强调效率 |

## 4. Wikipedia 维基百科

### 4.1 概述

Wikipedia（维基百科）是由吉米·威尔士于 **2001 年 1 月 15 日** 创立的多语言协作式免费百科全书。

### 4.2 关键信息

| 项目 | 内容 |
|------|------|
| 名称由来 | "wiki"（协作技术）+ "encyclopedia"（百科全书） |
| 运营组织 | 非营利组织 Wikimedia Foundation |
| 定位 | 全球访问量最大的参考网站之一 |
| 规模 | 300+ 语言版本，6000 万+ 文章 |

### 4.3 与 Wiki 的关系

> Wikipedia 是 Wiki 技术最著名的应用项目，名称由"wiki"与"encyclopedia"组合而成，体现了其开放协作的理念。

## 5. 主流 Wiki 系统

### 5.1 系统对比

| 系统 | 类型 | 技术栈 | 适用场景 | 特点 |
|------|------|--------|----------|------|
| **MediaWiki** | 开源 | PHP + MySQL | 技术文档 | 功能强大、可扩展、配置复杂 |
| **DokuWiki** | 开源 | 纯文本存储 | 中小团队 | 轻量、迁移简单、安全性高 |
| **Confluence** | 商业 | 云端/自部署 | 企业协作 | 与 Jira 深度集成、约 $5.75/用户/月 |
| **Notion** | 商业 | 云端 | 灵活协作 | 界面现代化、个人免费、团队 $8/用户 |

### 5.2 选型建议

| 需求 | 推荐系统 |
|------|----------|
| 技术文档首选 | MediaWiki 或 DokuWiki |
| 企业协作首选 | Confluence 或 Notion |

### 5.3 MediaWiki 详情

MediaWiki 是维基百科所使用的开源 Wiki 引擎：

- **高度可扩展性**：支持扩展插件生态系统
- **强大的版本控制**：完整的修订历史管理
- **多语言支持**：内置国际化功能
- **模板系统**：灵活的内容复用
- **用户群体**：全球数万个网站和数千家企业使用

## 7. 优势与挑战

### 7.1 Wiki 的优势

| 优势 | 说明 |
|------|------|
| 打破信息孤岛 | 集中存储，统一访问 |
| 跨部门知识关联 | 双向链接实现知识网络 |
| 双向链接 | 页面间相互引用，便于追溯 |
| 知识图谱构建 | 结构化知识组织 |
| 实时协作 | 多人同时编辑 |
| 版本控制 | 完整的变更历史 |

### 7.2 面临的挑战

| 挑战 | 说明 |
|------|------|
| 学习曲线较陡 | 新用户需要时间熟悉编辑规范 |
| 权限控制复杂 | 需要精细化的访问管理 |
| 内容质量控制 | 依赖社区参与和审核机制 |
